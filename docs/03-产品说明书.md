# 软件产品说明书

> 产品名称: SRE-agent — 面向麒麟操作系统的安全智能运维 Agent
> 版本: v0.2.0 | 日期: 2026-06-12

---

## 1. 产品概述

SRE-agent 是一套部署于麒麟操作系统（兼容通用 Linux）的智能运维 Agent，基于 MCP (Model Context Protocol) 协议构建，作为自然语言与 OS 交互的安全桥梁。系统通过将底层运维操作封装为标准化的 MCP Tool，使大语言模型能够在受控范围内感知系统状态、执行运维任务——**AI 永不直接接触 Shell**。

### 1.1 产品定位

| 维度 | 说明 |
|------|------|
| 目标用户 | 企业数据中心运维人员、系统管理员 |
| 应用场景 | 麒麟服务器集群的日常巡检、安全审计、故障诊断、性能监控 |
| 核心价值 | 用自然语言对话替代复杂命令行，同时通过多层安全护栏杜绝误操作 |

### 1.2 产品形态

B/S 架构 Web 应用，包含三个功能页面：对话交互、系统仪表盘、审计日志追溯。后端提供 REST API + SSE 流式接口，大模型通过 Ollama / DeepSeek / Qwen3 本地或远程部署。

## 2. 功能清单

### 2.1 MCP 运维工具 (45 个)

系统将底层 OS 命令封装为 45 个标准化的 MCP Tool，按功能分为 8 大类：

| 类别 | 数量 | 代表工具 | 风险等级 |
|------|:--:|---------|:--:|
| **进程感知** | 7 | `process_inspect`, `process_detail`, `process_tree`, `process_zombie_scan`, `process_top_cpu`, `process_top_memory`, `process_kill` | 6 只读 + 1 危险 |
| **磁盘感知** | 5 | `disk_inspect`, `disk_inode_usage`, `disk_io_stats`, `disk_mount_audit`, `disk_large_files` | 全部只读 |
| **网络感知** | 6 | `network_listening_ports`, `network_connections_summary`, `network_interface_stats`, `network_firewall_audit`, `network_tcp_retrans`, `network_dns_check` | 全部只读 |
| **内存感知** | 5 | `memory_info`, `swap_info`, `memory_oom_history`, `memory_hugepages`, `memory_slab_info` | 全部只读 |
| **安全态势** ⭐ | 11 | `security_auth_failures`, `security_active_sessions`, `security_suid_scan`, `security_crontab_audit`, `security_kernel_modules`, `security_pending_updates`, `security_user_audit`, `security_sysctl_audit`, `user_list`, `security_open_files`, `security_selinux_status` | 全部只读 |
| **系统健康** | 7 | `system_info`, `system_load`, `system_failed_services`, `system_boot_params`, `system_package_updates`, `system_entropy` | 全部只读 |
| **健康配置** | 2 | `health_config_get`, `health_config_set` | 1 只读 + 1 受限 |
| **容器感知** | 3 | `container_list`, `container_stats`, `container_inspect` | 全部只读 |

> 风险分布: 43 个 `read_only` (只读) + 1 个 `restricted` (需确认) + 1 个 `dangerous` (需管理员)

### 2.2 安全护栏体系

| 层级 | 模块 | 机制 |
|:--:|------|------|
| 第一道 | `intent_filter.py` v2.0 | 4 层递进意图分类 (越狱→高危→注入→运维)，威胁评分 0.0~1.0，词边界正则防误报 |
| 第二道 | `injection_detector.py` v2.0 | 7 层注入检测 (分隔符/编码绕过/间接注入/同形字/零宽字符/大小写混淆/LLM 输出校验) |
| 第三道 | `permission_agent.py` v2.0 | 三级 RBAC 权限 (ops_basic/ops_advanced/ops_admin)，sudo 降权执行，保护进程/路径名单 |
| 底层 | `_common.py` | 23 个命令白名单 + 3 级高危参数拦截 (词边界/精确 flag/S子串操作符) |

### 2.3 智能化根因分析

| 能力 | 实现 |
|------|------|
| 异常检测 | 4 种算法: 3-Sigma, IQR, 移动平均, 孤立森林，组合投票 (≥2 共识) |
| 健康评分 | 5 维度加权评分 0~100 (CPU 0.25 + 内存 0.25 + 磁盘 0.20 + 负载 0.15 + Swap 0.15)，A~F 等级 |
| 日志关联 | 系统日志结构化提取 + 关键词匹配 + 因果关系推断 |
| 阈值配置 | `health_config_get/set` 支持运行时调整评分配置，无需重启 |

### 2.4 前端交互

| 页面 | 功能 |
|------|------|
| ChatView (对话) | SSE 流式对话，Markdown 渲染，Tool 调用卡片，危险操作确认弹窗 |
| DashboardView (仪表盘) | CPU/内存/磁盘实时图表，进程 TOP 表，安全告警面板，系统概览 |
| AuditLogView (审计) | 5 阶段审计时间线，风险等级筛选，关键词搜索，详情展开 |

## 7. 安全声明

1. **AI 不接触 Shell**: 所有系统命令通过固定参数列表执行，LLM 仅能选择调用哪个 Tool 及传入 JSON 参数
2. **命令白名单**: 仅 23 个安全命令可通过 `run_command()` 执行，`rm`/`chmod`/`dd` 等危险命令不在白名单内
3. **高危参数拦截**: `-rf`/`&&`/`|`/`;` 等 shell 操作符在参数中会被自动拦截
4. **进程保护**: `systemd`/`sshd`/`kernel` 等 8 个关键进程及 PID 1 不可被 `process_kill` 终止
5. **路径保护**: `/etc/shadow`/`/boot`/`/sys`/`/proc` 等 10 个系统关键路径受保护
6. **最小权限**: 推荐创建专用 `sreagent` 系统用户，高危操作通过 `sudo -u sreagent` 降权执行
