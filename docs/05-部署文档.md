> SRE-agent 部署指南 — 麒麟 V10/V11 & 通用 Linux

## 1. 环境要求

| 依赖 | 最低版本 | 推荐版本 | 说明 |
|------|----------|----------|------|
| Python | 3.10 | 3.13 | psutil 需要 3.10+ |
| Node.js | 18 | 20 LTS | 前端构建 |

| 磁盘 | 2GB | 5GB+ | 含模型下载 |

## 2. 麒麟操作系统部署

### 2.1 环境准备

```bash
# 确认系统版本
cat /etc/os-release
# 应输出: ID="kylin" 或 PRETTY_NAME="Kylin V10"

# 安装 Python 3.13 (如系统自带版本过低)
sudo dnf install -y python3.13 python3.13-devel
sudo dnf install -y nodejs npm

# 确认架构 (龙芯需特殊注意)
uname -m
# loongarch64 或 x86_64
```

### 2.2 后端部署

```bash
cd backend

# 创建虚拟环境
python3.13 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 创建专用低权限用户 (安全最佳实践)
sudo useradd -r -s /bin/false -d /nonexistent -M sreagent

# 启动 (生产模式)
uvicorn app.main:app --host 0.0.0.0 --port 8001 --workers 4
```

### 2.3 前端部署

```bash
cd frontend

# 安装依赖
npm install

# 生产构建
npm run build

# 构建产物在 dist/ 目录, 可用 nginx 托管
sudo cp -r dist/* /var/www/html/sre-agent/
```

### 2.4 LLM 部署 (可选)

```bash
# 安装 Ollama
curl -fsSL https://ollama.com/install.sh | sh

# 拉取模型
ollama pull qwen3:4b

# 启动服务
ollama serve
```

### 2.5 Nginx 反向代理 (生产)

```nginx
server {
    listen 80;
    server_name sre-agent.local;

    # 前端静态文件
    location / {
        root /var/www/html/sre-agent;
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8001/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;

        # SSE 流式支持
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
    }
}
```

## 4. Docker 部署

```dockerfile
# Dockerfile
FROM python:3.13-slim

RUN apt-get update && apt-get install -y \
    nodejs npm \
    && rm -rf /var/lib/apt/lists/*

# 创建专用用户
RUN useradd -r -s /bin/false -d /nonexistent -M sreagent

WORKDIR /app

# 后端
COPY backend/requirements.txt .
RUN pip install -r requirements.txt
COPY backend/ .

# 前端
COPY frontend/ /frontend/
RUN cd /frontend && npm install && npm run build

EXPOSE 8001
CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]
```

```bash
# 构建与运行
docker build -t sre-agent .
docker run -d --name sre-agent \
  -p 8001:8001 \
  -v /var/log:/var/log:ro \
  -v /proc:/host-proc:ro \
  sre-agent
```

## 6. 安全配置建议

### 6.1 最小权限

```bash
# 1. 创建专用用户 (必须)
sudo useradd -r -s /bin/false -d /nonexistent -M sreagent

# 2. 配置 sudo 策略
sudo visudo -f /etc/sudoers.d/sre-agent
# 内容:
# %sreagent ALL=(sreagent) NOPASSWD: /usr/bin/systemctl status, /usr/bin/journalctl
```

### 6.2 日志保护

```bash
# 确保日志目录只读
sudo chmod 640 /var/log/auth.log
sudo chmod 640 /var/log/secure
```

### 6.3 网络隔离

```bash
# 生产环境建议仅监听本地
uvicorn app.main:app --host 127.0.0.1 --port 8001
# 通过 nginx 反向代理对外暴露
```

## 7. 故障排除

| 现象 | 原因 | 解决 |
|------|------|------|
| `psutil` 安装失败 | 缺少 python3-devel | `dnf install python3.13-devel` |
| `journalctl` 命令不存在 | 非 systemd 系统 | 自动降级到日志文件读取 |
| 龙芯架构 Tool 返回空 | psutil 版本过低 | `pip install psutil --upgrade` |
| 前端构建 OOM | Node 内存不足 | `NODE_OPTIONS=--max-old-space-size=4096 npm run build` |
| SSE 连接断开 | nginx 缓冲超时 | 添加 `proxy_buffering off; proxy_read_timeout 300s;` |
