## 2. 核心模块设计

### 2.1 安全护栏引擎 (`core/`) — 三道防线递进式防御

#### 2.1.1 `intent_filter.py` v2.0 — 意图风险分类器（第一道防线）

```
输入: 用户自然语言字符串
  │
  ├─ Layer 1: 越狱/角色劫持检测
  │   ├─ 11 个英文强模式正则 (\\b 词边界)
  │   │   e.g. "ignore (previous|above) instructions"
  │   │        "you are (now|no longer)", "pretend you are"
  │   │        "(dan|developer|god) mode", "jailbreak"
  │   │        "(bypass|override|disable) (security|safety|filter)"
  │   │        "switch (your )?role", "(system|new) (prompt|instructions?):"
  │   ├─ 7 个中文深层次越狱正则
  │   │   e.g. 角色扮演/催眠/编码绕过/DAN 模式中文版
  │   └─ 3 个弱信号组合检测 (≥2 → JAILBREAK)
  │       e.g. "tell me how to" + "hack|exploit|backdoor" + "no matter what"
  │
  ├─ Layer 2: 高危命令检测 (27 个危险模式, 全部 \\b 词边界)
  │   ├─ 文件系统破坏: rm -rf /etc|/var|/home..., mkfs, dd if=, >/dev/sdX
  │   ├─ 权限提升: chmod 777, chown -R, setfacl -m, chattr -i
  │   ├─ 网络危险: iptables -F, iptables -P ACCEPT, nft flush ruleset
  │   ├─ 系统破坏: shutdown/reboot/halt/poweroff, kill -9 -1|1
  │   ├─ 数据破坏: DROP TABLE/DATABASE, DELETE FROM, TRUNCATE
  │   ├─ 远程下载执行: wget|curl ... | bash, nc -e 反弹 shell
  │   └─ 内核/引导: grub/efibootmgr 修改, modprobe -r
  │   └─ 组合升级: ≥2 命中 → DANGEROUS_ACTION
  │
  ├─ Layer 3: 注入特殊字符检测 (15 个注入模式)
  │   ├─ 分隔符: ;/&&/|| + 危险命令
  │   ├─ 管道: | bash/| sh
  │   ├─ 命令替换: $(...), `...`, ${...}
  │   ├─ 重定向: > /绝对路径
  │   ├─ 续行隐藏: \\\n + 危险命令
  │   ├─ 执行函数: exec/eval/source /etc
  │   ├─ 编码绕过: ANSI-C 引用 ($'\\xNN...')
  │   ├─ 进程替换: <()/ >() 注入
  │   └─ heredoc 注入: <<EOF ... 危险命令
  │   └─ 组合升级: ≥3 命中 → DANGEROUS_ACTION
  │
  ├─ Layer 4: 运维操作关键词
  │   ├─ 中文运维动词 (1 组正则覆盖 11 个关键词)
  │   │   重启/停止/启动/禁用/启用/清空/卸载/安装/修改/删除/终止
  │   └─ 英文运维动词 (1 组正则覆盖 9 个关键词, \\b 词边界)
  │       restart/stop/start/disable/enable/remove/uninstall/kill/clean/purge
  │
  └─ 输出: (IntentCategory, hits[], threat_score 0.0~1.0)
```

**评分函数 `_calculate_threat_score()`**:

| 条件 | 威胁评分 | 等级 |
|------|---------|------|
| Layer 1 命中 | 1.0 | CRITICAL |
| Layer 2 命中 ≥2 | 0.9 | CRITICAL |
| Layer 2 命中 ×1 | 0.7 | HIGH |
| Layer 3 命中 ≥3 | 0.6 | MEDIUM |
| Layer 3 命中 ×1 | 0.3 | MEDIUM |
| Layer 4 命中 | 0.2 | LOW |
| 无命中 | 0.0 | SAFE |

**意图分类枚举 (IntentCategory)**:
- `jailbreak` — 直接拒绝, 不进入 LLM
- `dangerous_action` — 拦截并告警
- `ops_action` — 运维操作, 需二次确认
- `safe_query` — 安全查询, 直接放行
- `unknown` — 未分类

#### 2.1.2 `injection_detector.py` v2.0 — Prompt 注入检测器（第二道防线）

```
输入: 用户输入 (原始字符串, 保留大小写和 Unicode)
  │
  ├─ Layer 1: 分隔符注入
  │   ├─ 换行/回车后跟随危险命令 (\n|\r + rm/wget/curl/bash/sh/nc/...)
  │   └─ 空字节截断检测 (\x00)
  │
  ├─ Layer 2: 编码绕过
  │   ├─ Base64 解码 (匹配 ≥20 chars 候选串, 解码后检测危险内容)
  │   └─ Hex 转义序列 (\\xNN 连续 4+ 次)
  │
  ├─ Layer 3: 间接指令注入 (6 组中英文正则)
  │   ├─ 中文角色切换: "从现在开始...你是/你的身份是"
  │   ├─ 任务嵌套: "翻译/总结/复述以下...rm/wget/curl/..."
  │   ├─ 中文规则绕过: "忽略...规则/限制/安全...执行/运行/输出"
  │   ├─ 英文任务嵌套: "translate/summarize/repeat following..."
  │   ├─ 英文规则绕过: "ignore all rules/constraints/instructions"
  │   └─ 强制命令输出: "output only the command/code/script"
  │
  ├─ Layer 4: Unicode 同形字检测 (v2.0 新增)
  │   ├─ 87 条目同形字映射表:
  │   │   ├─ 拉丁扩展 (24 个): à→a, è→e, ì→i, ò→o, ù→u ...
  │   │   ├─ 西里尔 (9 个): а→a, е→e, о→o, р→p, с→c, у→y, х→x, і→i, ј→j
  │   │   ├─ 希腊 (6 个): ο→o, ν→v, α→a, ε→e, ι→i, κ→k
  │   │   └─ 全角拉丁 (52 个): ａ→a ... ｚ→z, Ａ→A ... Ｚ→Z
  │   ├─ 非 ASCII 字符检测 → 标准化为 ASCII
  │   └─ 14 个危险关键词匹配 (rm/wget/curl/bash/sh/chmod/sudo/kill/nc/...)
  │
  ├─ Layer 5: 零宽字符隐蔽注入 (v2.0 新增)
  │   ├─ 9 种零宽字符: U+200B/U+200C/U+200D/U+FEFF/U+2060~U+2064
  │   └─ 9 种 Bidi 覆盖字符: LRE/RLE/PDF/LRO/RLO/LRI/RLI/FSI/PDI
  │
  ├─ Layer 6: 大小写交替混淆 (v2.0 新增)
  │   ├─ aAaA... 模式 (4+ chars 交替)
  │   ├─ AaAa... 模式 (4+ chars 交替)
  │   ├─ 2-char 交替对 (如 rM, rM -rF)
  │   └─ 17 个危险命令小写比对 (rm/wget/curl/bash/chmod/chown/sudo/kill/...)
  │
  └─ Layer 7: LLM 输出黑名单校验
      ├─ rm -rf /, dd if=, >/dev/sdX, mkfs., chmod 777
      ├─ Fork 炸弹 (:(){ :|:& };:)
      └─ wget/curl -O 覆盖系统文件 (/etc/)
```

**安全流水线 `safe_pipeline()`**:
```
用户输入
  → ① classify_intent()     — 意图拦截 (JAILBREAK / DANGEROUS 不进 LLM)
  → ② detect_injection()    — 注入拦截 (7 层检测, 任一命中 → BLOCK)
  → ③ validate_llm_output() — 输出校验 (防止 LLM 意外输出危险命令)
  → PASS / OPS_CONFIRM / BLOCK
```

#### 2.1.3 `permission_agent.py` v2.0 — 最小权限执行代理（第三道防线）

```
风险等级 → 所需权限映射:
  read_only  → ops_basic     (任何人可执行, 包括 sandbox 用户)
  restricted → ops_advanced  (sudo 组成员, 需二次确认)
  dangerous  → ops_admin     (仅 root, 需管理员授权)

权限层级 (从高到低):
  root (ops_admin)
    → sudo 组成员 (ops_advanced)
      → sreagent 专用用户 (ops_basic+, 建议部署时创建)
        → nobody 系统用户 (ops_basic, fallback)

sudo 降权机制:
  build_sudo_command(cmd) → ["sudo", "-u", "sreagent", "--"] + cmd
  高危操作自动降权到低权限用户执行, 确保攻击者仅获得受限身份
```

**保护名单 (最高优先级, 即使 root 也不可操作)**:

| 类别 | 数量 | 内容 |
|------|------|------|
| 受保护进程 | 10 个 | systemd, sshd, kernel, init, dbus-daemon, journald, auditd, rsyslogd, cron, agetty |
| 受保护路径 | 10 个 | /etc/shadow, /etc/passwd, /boot, /sys, /proc, /etc/sudoers, /etc/sudoers.d, /etc/ssh/sshd_config, /etc/pam.d, /etc/selinux |
| 受限写目录 | 9 个 | /etc, /usr/bin, /usr/sbin, /usr/lib, /usr/lib64, /lib, /lib64, /bin, /sbin |

**关键函数**:

| 函数 | 功能 |
|------|------|
| `check_permission(risk_level, user, target, action)` | 操作前权限预检, 返回 `(allowed, reason, downgrade_user)` |
| `build_sudo_command(cmd_list, target_user)` | 构建 sudo 降权命令 |
| `validate_path(path, action)` | 路径安全性校验 (规范化 + 保护名单 + 受限写目录) |
| `require_confirmation(risk_level)` | 判断是否需要用户二次确认 |
| `get_permission_level()` | 返回当前用户权限等级 |
| `permission_summary()` | 返回权限状态摘要 (用于仪表盘) |
| `setup_instructions()` | 返回 sreagent 用户创建指引 |

### 2.2 RCA 根因分析引擎 (`core/rca_analyzer.py`)

```
输入: plugin_data (来自 MCP Tool 调用结果)
  │
  ├─ 1. 异常检测 — 4 种算法并行 + 组合投票
  │   ├─ ThreeSigmaDetector (3-Sigma)
  │   │   └─ 均值 ± 3σ, 适用于正态分布数据 (需 ≥3 数据点)
  │   ├─ IQRDetector (四分位距)
  │   │   └─ Q3−Q1 × 1.5, 适用于偏态分布数据 (需 ≥4 数据点)
  │   ├─ MovingAverageDetector (移动平均)
  │   │   └─ 窗口=5, 偏离因子=2.0, 含偏离百分比 (需 ≥window+1 点)
  │   └─ IsolationForestDetector (孤立森林)
  │       └─ 多维联合检测 (CPU+内存+磁盘), sklearn 不可用时自动降级
  │
  │   组合投票 detect_anomaly_combined():
  │     ≥2 算法共识 → HIGH 置信度
  │     输出: consensus_anomalies, consensus_count, confidence
  │
  ├─ 2. 日志关联
  │   ├─ extract_log_events() — 结构化解析
  │   │   ├─ syslog 格式: MMM DD HH:MM:SS host process[pid]: message
  │   │   ├─ journalctl 格式: process[pid]: message
  │   │   ├─ 严重程度提取: EMERG→DEBUG (8 级)
  │   │   └─ 关键词提取: 10 种模式 (OOM/崩溃/磁盘满/超时/网络/权限/认证/CPU锁死/进程被杀/任务阻塞)
  │   └─ build_event_timeline() — 按来源分组时间线
  │
  ├─ 3. 因果推断 infer_causal_chain()
  │   ├─ 7 组因果匹配模式 (前因-后果正则对 + 置信度):
  │   │   ├─ OOM Killer → 进程被终止 (0.95)
  │   │   ├─ 磁盘空间耗尽 → 写入失败 (0.90)
  │   │   ├─ CPU 锁死 → 任务阻塞 (0.90)
  │   │   ├─ 进程崩溃 → 服务停止 (0.88)
  │   │   ├─ 网络中断 → 连接失败 (0.85)
  │   │   ├─ 暴力破解 → 账户锁定 (0.82)
  │   │   └─ DNS 解析失败 → 服务访问异常 (0.80)
  │   ├─ 根因识别: 是前因但不是后果的事件
  │   └─ 受影响服务识别: 是后果但不是前因的事件
  │
  ├─ 4. 健康评分 compute_health_score()
  │   ├─ 5 维度加权评分:
  │   │   ├─ CPU 使用率: 0.25
  │   │   ├─ 内存使用率: 0.25
  │   │   ├─ 磁盘使用率: 0.20
  │   │   ├─ 系统负载比: 0.15 (load / cores)
  │   │   └─ Swap 使用率: 0.15
  │   │
  │   ├─ 分级阈值 (可配置, 通过 health_config_set 调整):
  │   │   e.g. CPU: <50%=100分, <70%=80分, <85%=60分, <95%=30分, ≥95%=10分
  │   │   e.g. 内存: <60%=100分, <80%=80分, <90%=50分, <95%=25分, ≥95%=5分
  │   │
  │   ├─ 评级: A(≥90) / B(≥75) / C(≥60) / D(≥40) / F(<40)
  │   │
  │   ├─ 告警生成: 每维度独立告警规则
  │   │   e.g. 内存: score≤50→"内存使用率偏高", ≤25→"内存不足", ≤5→"内存耗尽"
  │   │

  │       load_health_config() / save_health_config() 读写校验
  │
  └─ 输出: run_rca_analysis() 完整 RCA 报告
      { timestamp, sections: { anomaly_detection, log_correlation,

### 3.1 安全查询流程

```

  │
  ▼

      ├─ check_permission("read_only") → (allowed=True, "", None)
      └─ system_load_handler() → run_command(["cat", "/proc/loadavg"])
  │
  ▼
LLM 接收 Tool 结果 → 生成自然语言回复 (中文)
  │
  ▼
_extract_metrics() + compute_health_score()

  │
  ▼

  token → token → ... → token
  → tool_call {system_load}
  → tool_result {system_load 结果}

  │

  │
  ▼

| `done` | 对话流正常结束 | `{}` | 设置 streaming=false, 清理 |
| `error` | 安全拦截/LLM 异常 | `{message: string}` | 追加错误到消息内容 |
