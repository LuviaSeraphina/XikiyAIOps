## 2. 核心模块设计

### 2.1 安全护栏引擎 (`core/`)

#### 2.1.1 intent_filter.py — 意图风险过滤器

```
输入: 用户自然语言
  │
  ├─ Layer 1: 越狱/角色劫持检测
  │   ├─ 11 个英文强模式正则 (词边界)
  │   ├─ 7 个中文深层次越狱正则
  │   └─ 3 个弱信号组合检测 (≥2 → JAILBREAK)
  │
  ├─ Layer 2: 高危命令检测
  │   ├─ 27 个危险模式正则 (全部 \\b 词边界)
  │   └─ 组合升级: ≥2 命中 → DANGEROUS_ACTION
  │
  ├─ Layer 3: 注入字符检测
  │   └─ 12 个注入模式 (分隔符/管道/重定向/eval/exec)
  │
  ├─ Layer 4: 运维操作关键词
  │   ├─ 中文运维动词 (10个)
  │   └─ 英文运维动词 (9个)
  │
  └─ 输出: (IntentCategory, hits[], threat_score)
```

**v2.0 关键改进**:
- 全部正则使用 `\b` 词边界, 消除如 "start" 匹配 "restart" 的误报
- 组合信号加权评分 (`_calculate_threat_score`)
- `get_threat_level()` 返回 0.0~1.0 威胁评分 + CRITICAL/HIGH/MEDIUM/LOW/SAFE 等级

#### 2.1.2 injection_detector.py — Prompt 注入检测器

```
输入: 用户输入 (原始字符串)
  │
  ├─ Layer 1: 分隔符注入 (\\n, \\r, \\x00)
  ├─ Layer 2: 编码绕过 (Base64 解码 + Hex 转义)
  ├─ Layer 3: 间接指令注入 (翻译嵌套/规则绕过)
  ├─ Layer 4: Unicode 同形字混淆 (v2.0 NEW)
  │   └─ 西里尔/希腊 → ASCII 标准化 + 危险关键词匹配
  ├─ Layer 5: 零宽字符隐蔽注入 (v2.0 NEW)
  │   └─ 9 种零宽字符 + 9 种 Bidi 覆盖字符检测
  ├─ Layer 6: 大小写交替混淆 (v2.0 NEW)
  │   └─ aAaA / AaAa 模式 + 危险命令小写比对
  │
  └─ 输出: list[hits]
```

#### 2.1.3 permission_agent.py — 最小权限执行代理

```
风险等级映射:
  read_only  → ops_basic    (任何人)
  restricted → ops_advanced (sudo 组成员)
  dangerous  → ops_admin    (仅 root)

权限层级:
  root (ops_admin) → sudo组 (ops_advanced) → sreagent (ops_basic+) → nobody (ops_basic)

sudo 降权:
  build_sudo_command(cmd) → ["sudo", "-u", "sreagent", "--"] + cmd
```

**v2.0 关键改进**:
- 专用低权限用户 `sreagent` (部署时通过 `setup_instructions()` 引导创建)
- `build_sudo_command()` 自动将高危命令包装为低权限执行
- `check_permission()` 返回降权建议用户
- 保护名单扩展至 10 个进程 + 9 个路径 + 9 个受限写目录

### 2.2 RCA 根因分析引擎 (`core/rca_analyzer.py`)

```
输入: plugin_data (来自各 MCP Tool)
  │
  ├─ 1. 异常检测
  │   ├─ 3-Sigma (正态分布数据)
  │   ├─ IQR (偏态分布数据)
  │   ├─ 移动平均 (趋势数据)
  │   └─ 组合投票 (≥2 算法共识)
  │
  ├─ 2. 日志关联
  │   ├─ 事件结构化提取 (syslog/journalctl 解析)
  │   ├─ 严重程度分级 (EMERG→DEBUG)
  │   └─ 关键词提取 (OOM/崩溃/磁盘满/...)
  │
  ├─ 3. 因果推断
  │   ├─ 7 组因果匹配模式 (正则对)
  │   ├─ 时间顺序验证
  │   └─ 根因链 + 受影响服务识别
  │
  ├─ 4. 健康评分
  │   ├─ 5 维度加权: CPU(0.25) + 内存(0.25) + 磁盘(0.20) + 负载(0.15) + Swap(0.15)
  │   └─ 评级: A(90+) → F(<40)
  │
  └─ 输出: 完整 RCA 报告
```

### 2.3 MCP 插件注册中心 (`mcp_plugins/base.py`)

```
MCPToolRegistry (单例):
  register(tool)     → 注册 Tool
  get_tool(name)     → 按名称获取
  list_all()         → 列出所有 Schema
  list_by_risk(max)  → 按风险过滤
  call(name, **kw)   → 统一调用 (含权限预检)
  count()            → 统计数量

MCPTool:
  name / description / handler / risk_level / parameters
  to_schema()        → MCP 协议 JSON Schema
  execute(**kw)      → 执行 handler
```

### 2.4 命令安全执行 (`mcp_plugins/_common.py`)

```
run_command(cmd):
  1. _is_safe_command(cmd) → 白名单校验
     - 基础命令: 16 个 (ss, ip, who, find, journalctl, systemctl, ...)
     - 高危参数: 16 个 (rm, -rf, mkfs, dd, shutdown, ;, &&, |, ...)
  2. subprocess.run(cmd, timeout=10)
  3. 返回 stdout (失败返回空字符串, 永不崩溃)
```

## 3. 数据流设计

### 3.1 用户查询流程

```
用户输入 "查看系统负载"
  → ① safe_pipeline() 安全流水线
     ├─ classify_intent() → SAFE_QUERY (score=0.0)
     ├─ detect_injection() → []
     └─ → PASS
  → ② LLM 解析意图 → 决定调用 system_load tool
  → ③ registry.call("system_load")
     ├─ check_permission("read_only") → (True, "", None)
     └─ system_load_handler() 执行
  → ④ 返回结果给 LLM 生成自然语言回复
  → ⑤ validate_llm_output() 二次校验
  → ⑥ SSE 流式推送给前端
```

### 3.2 高危操作拦截流程

```
用户输入 "rm -rf /etc/nginx"
  → ① classify_intent() → DANGEROUS_ACTION (score=0.9)
     hits: ["DANGEROUS: 递归强制删除 (rm -rf)"]
  → ② 直接拦截, 不进入 LLM
  → ③ 返回: "操作被安全护栏拦截: 递归强制删除"
  → ④ 记录审计日志
```

### 4.2 技术栈

| 层 | 选型 | 原因 |
|----|------|------|
| 框架 | Vue 3 (Composition API) | 轻量, 适合比赛项目 |
| UI 库 | Element Plus | 企业级组件, 中文友好 |
| 图表 | ECharts 5 | 仪表盘可视化 |
| 状态 | Pinia | Vue 3 官方推荐 |
| 路由 | Vue Router 4 | SPA 路由 |
| 流式 | SSE (Server-Sent Events) | 单向流, 比 WebSocket 简单 |
| Markdown | marked + highlight.js | 回复渲染 + 代码高亮 |
