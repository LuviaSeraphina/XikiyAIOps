# 需求分析文档

> 第十五届中国软件杯大赛 A 组赛题 — 面向麒麟操作系统的安全智能运维 Agent
> 出题企业: 麒麟软件有限公司

---

## 1. 功能需求

### 1.1 系统感知 — MCP Tools (45 个，7 大类)

MCP Tool 层是系统的核心感知与操作接口，封装所有底层系统命令为具有统一安全协议的原子操作。每项 Tool 声明 `risk_level` (read_only / restricted / dangerous)，参数通过 `inputSchema` JSON Schema 严格校验。

#### 进程感知 (process — 7 个)
- **FR-PROC-01** `process_inspect`: 进程列表查看，支持按状态过滤、按 CPU/内存排序
- **FR-PROC-02** `process_detail`: 单进程详情 (PID/名称/状态/CPU/内存/线程数/FD/运行时间/工作目录/可执行文件/用户/nice 值)
- **FR-PROC-03** `process_tree`: 构建进程树，以 PID 1 为根展示父子关系
- **FR-PROC-04** `process_zombie_scan`: 僵尸进程扫描，自动告警
- **FR-PROC-05** `process_top_cpu`: CPU 占用 Top N (可配置 1~20，默认 5)
- **FR-PROC-06** `process_top_memory`: 内存占用 Top N (含 RSS 物理内存)
- **FR-PROC-07** `process_kill` (dangerous): 终止指定进程 (SIGTERM/SIGKILL)，多级护栏阻止终止 init/自身/关键系统进程

#### 磁盘感知 (disk — 5 个)
- **FR-DISK-01** `disk_inspect`: 磁盘空间使用情况，支持指定挂载点
- **FR-DISK-02** `disk_inode_usage`: Inode 使用率检测 (生产故障高频根因: inode 耗尽但磁盘有空间)
- **FR-DISK-03** `disk_io_stats`: 磁盘 I/O 统计 (读写次数/吞吐量/耗时)
- **FR-DISK-04** `disk_mount_audit`: 挂载点安全审计 (noexec/nosuid/ro 属性检查)
- **FR-DISK-05** `disk_large_files`: 大文件扫描，按大小降序返回

#### 网络感知 (network — 6 个)
- **FR-NET-01** `network_listening_ports`: 监听端口审计 (TCP/UDP，含进程名和 PID)
- **FR-NET-02** `network_connections_summary`: TCP 连接状态统计 (CLOSE_WAIT/SYN_SENT 异常告警)
- **FR-NET-03** `network_interface_stats`: 网卡流量与错误统计 (丢包/错误检测)
- **FR-NET-04** `network_firewall_audit`: 防火墙规则审计 (nftables/iptables 自动检测)
- **FR-NET-05** `network_tcp_retrans`: TCP 重传率检测 (>2% 告警)
- **FR-NET-06** `network_dns_check`: DNS 解析测试 (dig/getent 兜底，支持指定 DNS 服务器)

#### 内存感知 (memory — 5 个)
- **FR-MEM-01** `memory_info`: 物理内存画像 (总量/已用/可用/使用率)
- **FR-MEM-02** `swap_info`: Swap 交换分区画像 (使用率 >50% 预警)
- **FR-MEM-03** `memory_oom_history`: OOM Killer 历史追溯 (journalctl/dmesg)
- **FR-MEM-04** `memory_hugepages`: 大页内存状态 (总量/空闲/预留/盈余/页面大小)
- **FR-MEM-05** `memory_slab_info`: 内核 Slab 缓存用量 (可回收/不可回收)

#### 安全态势 (security — 11 个) ⭐
- **FR-SEC-01** `security_auth_failures`: 多源认证失败统计 (SSH/su/sudo/PAM)，含 fail2ban 封禁信息
- **FR-SEC-02** `security_active_sessions`: 活跃登录会话和 SSH 连接枚举
- **FR-SEC-03** `security_suid_scan`: SUID/SGID 后门文件扫描，标记非白名单文件
- **FR-SEC-04** `security_crontab_audit`: 全用户 crontab 持久化审计 (检测 curl/wget 下载脚本等可疑任务)
- **FR-SEC-05** `security_kernel_modules`: 已加载内核模块审计 (Rootkit 检测)
- **FR-SEC-06** `security_pending_updates`: 待安装安全更新检测 (dnf/apt 自动适配)
- **FR-SEC-07** `security_user_audit`: 用户权限审计 (空密码/UID=0/NOPASSWD sudo)
- **FR-SEC-08** `security_sysctl_audit`: 内核安全参数基线审计 (ASLR/ptrace/IP 转发等 12 项)
- **FR-SEC-09** `user_list`: 用户与组列表查询 (UID/GID/家目录/Shell)
- **FR-SEC-10** `security_open_files`: 打开文件数 Top N (FD >1000 时告警，句柄泄漏检测)
- **FR-SEC-11** `security_selinux_status`: SELinux/AppArmor 运行模式检测 (均未启用时告警)

#### 系统健康 & 配置 (system — 8 个)
- **FR-SYS-01** `system_info`: 系统概览 (主机名/内核/发行版/架构/运行时间，含麒麟平台检测)
- **FR-SYS-02** `system_load`: 系统负载 (1/5/15 分钟)，含 CPU 核心数感知的过载判断
- **FR-SYS-03** `system_failed_services`: 失败 systemd 服务检测 (systemctl --failed)
- **FR-SYS-04** `system_boot_params`: 内核启动参数安全审计
- **FR-SYS-05** `system_package_updates`: 待安装安全更新数量 (dnf/apt 自动适配)
- **FR-SYS-06** `system_entropy`: 内核熵池可用量 (低熵 <500 影响 TLS/SSH 等加密服务)
- **FR-SYS-07** `health_config_get` (read_only): 获取健康评分权重和阈值配置
- **FR-SYS-08** `health_config_set` (restricted): 修改健康评分阈值或权重 (dot-path 参数)

#### 容器感知 (container — 3 个)
- **FR-CTR-01** `container_list`: Docker/Podman 容器列表 (名称/镜像/状态/端口/运行时长)
- **FR-CTR-02** `container_stats`: 容器资源用量 (CPU%/内存/网络 I/O)
- **FR-CTR-03** `container_inspect`: 单容器安全审计 (privileged/Capabilities/挂载卷/环境变量)

### 1.2 安全护栏体系 (7 道检测层 + 4 层意图过滤 + 权限代理)

系统的安全护栏采用纵深防御策略，v2.0 版本形成了完整的 7 层注入检测 + 4 层意图过滤 + 最小权限代理体系。

#### 1.2.1 意图过滤器 (intent_filter v2.0 — 4 层递进分类)

```
Layer 1 (越狱检测) —— 一击即杀，最高优先级
    ├── 英文指令劫持: ignore previous instructions / you are now / pretend you are / DAN mode
    ├── 中文深层次越狱: 忘记之前指令 / 从现在起你是 / 角色扮演终端 / 催眠模式
    └── 组合越狱信号: ≥2 个弱信号 (hack + no restriction) → JAILBREAK
    └── 词边界正则 (\b) 消除误匹配 (如 "start" 不再匹配 "restart")

Layer 2 (高危命令检测) —— 词边界正则 + 组合加权
    ├── 文件系统破坏: rm -rf /etc /var /home, mkfs, dd, fdisk
    ├── 权限提升/破坏: chmod 777, chown -R, setfacl, chattr
    ├── 网络危险操作: iptables -F, nft flush, ufw disable
    ├── 系统破坏: shutdown/reboot/halt, kill -9 -1
    ├── 数据破坏: DROP TABLE, DELETE FROM, TRUNCATE
    ├── 远程下载执行: wget/curl | bash, nc -e 反弹 shell
    └── 组合升级: ≥2 条命中 → DANGEROUS_ACTION

Layer 3 (注入特殊字符) —— 命令注入与特殊字符检测
    ├── 分隔符注入: ; && || 后接危险命令
    ├── 管道注入: | bash, | sh
    ├── 命令替换: $(), `backticks`, ${}
    ├── 重定向注入: > /absolute/path
    ├── 高级绕过: \换行续行, exec, eval, source
    ├── ANSI-C 引用: $'\\x...' 编码绕过
    └── 进程替换/heredoc: <() / >() / <<EOF
    └── 组合升级: ≥3 条命中 → DANGEROUS_ACTION

Layer 4 (运维操作关键词) —— 需二次确认
    ├── 中文: 重启/停止/启动/禁用/启用/清空/卸载/安装/修改/删除/终止
    └── 英文: restart/stop/start/disable/enable/remove/uninstall/kill/clean
```

- **FR-SAFE-01**: 四层递进分类 (Layer 1 → Layer 2 → Layer 3 → Layer 4)，短路返回 (Layer 1/2 直接拦截)
- **FR-SAFE-02**: 词边界正则匹配 (\b) 消除子串误匹配 + 组合加权评分
- **FR-SAFE-03**: 中英文双语越狱模式检测 (7 类中文深层次越狱模式)
- **FR-SAFE-04**: 威胁置信度评分 0.0~1.0 + 风险等级 (CRITICAL/HIGH/MEDIUM/LOW/SAFE)

#### 1.2.2 注入检测器 (injection_detector v2.0 — 7 层检测)

| 层 | 名称 | 检测能力 | 典型攻击 |
|----|------|---------|---------|
| Layer 1 | 分隔符注入 | \n, \r, \x00 隐藏指令 | "请帮我查进程\nrm -rf /" |
| Layer 2 | 编码绕过 | Base64/Hex 解码后匹配 | echo "cm0gLXJmIC8=" \| base64 -d |
| Layer 3 | 间接指令注入 | 翻译/总结/复述嵌套 | "翻译以下文字: rm -rf /" |
| Layer 4 | 同形字混淆 ⭐ | 西里尔/希腊/全角→ASCII 归一化 | "rⅿ -rf /" (ⅿ=U+217F) |
| Layer 5 | 零宽字符注入 ⭐ | U+200B/U+200C/U+200D/U+FEFF | 零宽字符分隔命令关键词 |
| Layer 6 | 大小写交替 ⭐ | 检测 rM/Rm/ChMoD 等模式 | "rM -Rf /etc" |
| Layer 7 | LLM 输出校验 | 输出黑名单二次扫描 | LLM 意外输出 rm -rf / |

- **FR-SAFE-05**: 分隔符注入检测 (\n/\r/\x00 + 空字节截断)
- **FR-SAFE-06**: 编码绕过检测 (Base64 解码匹配 + Hex 转义序列)
- **FR-SAFE-07**: 间接指令注入检测 (任务嵌套/规则绕过/强制输出命令)
- **FR-SAFE-08**: Unicode 同形字混淆检测 (拉丁/西里尔/希腊/全角 → ASCII 归一化 + 危险关键词匹配)
- **FR-SAFE-09**: 零宽字符隐蔽注入检测 (9 类零宽字符 + 双向文本覆盖 Bidi override)
- **FR-SAFE-10**: 大小写交替混淆检测 (4+ 字符连续交替 + 2 字符短模式如 rM)
- **FR-SAFE-11**: LLM 输出黑名单二次校验 (禁止 rm -rf /, dd, mkfs, chmod 777, fork bomb 等 7 类模式)

#### 1.2.3 最小权限代理 (permission_agent v2.0)

| 风险等级 | 所需权限 | 默认行为 | 降权策略 |
|---------|---------|---------|---------|
| `read_only` | ops_basic (所有人) | 直接放行 | 无 |
| `restricted` | ops_advanced (sudo/wheel 组) | 二次确认 + 建议降权 | sudo -u sreagent |
| `dangerous` | ops_admin (仅 root) | 管理员授权确认 | sudo -u sreagent (root 执行时) |

- **FR-SAFE-12**: 风险等级 → 权限等级映射 (read_only→ops_basic, restricted→ops_advanced, dangerous→ops_admin)
- **FR-SAFE-13**: sudo 降权执行机制 (高危操作强制降权到 sreagent 用户)
- **FR-SAFE-14**: 专用低权限系统用户 `sreagent` (无登录权限/无家目录)，部署时创建
- **FR-SAFE-15**: 命令白名单 (23 个允许命令: ss/ip/who/find/lsmod/systemctl/journalctl/dmesg/cat/crontab/sysctl/docker/podman/which/dnf/yum/apt/iptables/nft/getenforce/aa-status/dig/getent) + 高危参数拦截 (rm/mkfs/dd/shutdown/reboot/poweroff/halt/chmod/chown)
- **FR-SAFE-16**: 保护进程名单 (systemd/sshd/kernel/init/journald/auditd/cron 等 10 项) + 保护路径名单 (/etc/shadow, /etc/passwd, /boot, /sys, /proc, /etc/sudoers 等 9 项) + 受限写路径 (/etc, /usr/bin, /usr/sbin, /lib 等 9 项)

### 1.3 智能化根因分析 (RCA Engine)

RCA (Root Cause Analysis) 引擎提供 4 种异常检测算法 + 日志关联 + 因果推断 + 健康评分的完整链路。

#### 1.3.1 异常检测 (4 种算法，无需训练数据)

| 算法 | 类 | 原理 | 适用场景 |
|------|----|------|---------|
| 3-Sigma | `ThreeSigmaDetector` | 均值 ± 3σ 动态阈值 | 正态分布指标 (负载/CPU) |
| IQR 箱线图 | `IQRDetector` | Q1-1.5×IQR ~ Q3+1.5×IQR | 偏态分布/有离群值 |
| 移动平均 | `MovingAverageDetector` | 滑动窗口 + 偏离因子 | 时序趋势检测 |
| 孤立森林 | `IsolationForestDetector` | 多维联合异常 (CPU+内存+磁盘) | 多维关联异常 |

- **FR-RCA-01**: 3-Sigma 动态阈值异常检测 (均值/标准差/上下界/异常索引)
- **FR-RCA-02**: IQR 箱线图异常检测 (Q1/Q3/IQR/上下界，multiplier=1.5)
- **FR-RCA-03**: 移动平均异常检测 (滑动窗口 window=5, threshold_factor=2.0)
- **FR-RCA-04**: 多算法组合投票 (`detect_anomaly_combined`，≥2 算法共识)
- **FR-RCA-05**: 日志事件结构化提取 (`extract_log_events`，severity/source/关键词)
- **FR-RCA-06**: 因果关联模式匹配 (OOM/磁盘/网络/CPU/认证 5 类因果模式)
- **FR-RCA-07**: 因果链推断 (`infer_causal_chain`，置信度 ≥0.7)
- **FR-RCA-08**: 系统健康度综合评分 `compute_health_score` (0~100，5 维度加权 + 字母评级)

#### 1.3.2 健康评分体系 (5 维度加权)

| 维度 | 权重 | 评分规则 (0~100) | A-D-F 评级 |
|------|:----:|---------|------------|
| CPU | 0.25 | ≤50%=100, ≤70%=80, ≤85%=60, ≤95%=30, \>95%=10 | A(≥90), B(≥75), C(≥60), D(≥40), F(<40) |
| 内存 | 0.25 | ≤60%=100, ≤80%=80, ≤90%=50, ≤95%=25, \>95%=5 | |
| 磁盘 | 0.20 | ≤60%=100, ≤80%=80, ≤90%=50, ≤95%=20, \>95%=5 | |
| 负载 | 0.15 | ≤0.7x=100, ≤1.0x=80, ≤2.0x=50, ≤5.0x=20, \>5.0x=5 | |
| Swap | 0.15 | ≤20%=100, ≤50%=70, ≤80%=40, \>80%=10 | |

- **FR-HLTH-01**: 配置文件驱动 (`health_score_config.json`)，权重可在线调整
- **FR-HLTH-02**: 评分结果含字母评级 (A/B/C/D/F) + 分维度得分 + 告警列表
- **FR-HLTH-03**: `health_config_set` 可在运行时通过 dot-path 修改阈值/权重

## 2. 非功能需求

### 2.1 安全需求 (核心底线)

| 编号 | 需求描述 | 实现方式 |
|------|---------|---------|
| **NFR-SEC-01** | AI 永远不直接接触 Shell | 所有命令经 `registry.call()` 统一入口，由 handler 内部调用 `_common.run_command()` |
| **NFR-SEC-02** | 所有命令执行必须通过 MCP Tool 层 | MCPPluginRegistry 单例注册中心，无直接系统调用 |
| **NFR-SEC-03** | 高危操作必须记录审计日志 | 五阶段审计 (输入→感知→推理→校验→执行)，SQLite 持久化 |
| **NFR-SEC-04** | 命令参数必须是固定列表，禁止字符串拼接 | 命令白名单 (23 条) + 参数只允许固定枚举值 |
| **NFR-SEC-05** | 禁止危险操作暴露为 Tool | `rm -rf`, `chmod 777`, `iptables -F`, `nft flush ruleset` 等不暴露或仅 read_only 检查 |
| **NFR-SEC-06** | 命令白名单 + 高危参数拦截 | `_common.py` 中 `_ALLOWED_COMMANDS` (23 条) + `_DANGEROUS_WORD`/`_DANGEROUS_FLAG`/`_DANGEROUS_SUBSTR` 三层拦截 |

### 2.2 兼容性需求

- **NFR-COMPAT-01**: 支持麒麟 V10 (x86_64 + 龙芯 LoongArch)
- **NFR-COMPAT-02**: 支持麒麟 V11
- **NFR-COMPAT-03**: 兼容 Ubuntu 22.04 / Debian 12 等通用 Linux 发行版
- **NFR-COMPAT-04**: 自动检测包管理器 (`/usr/bin/dnf` 优先 → `apt` 兜底)
- **NFR-COMPAT-05**: 自动检测防火墙类型 (`/usr/sbin/nft` → nftables, 否则 iptables)
- **NFR-COMPAT-06**: 自动检测 SELinux 状态 (`/sys/fs/selinux` 存在性)
- **NFR-COMPAT-07**: Docker/Podman 容器运行时自动适配 (`docker` 优先 → `podman` 兜底)
- **NFR-COMPAT-08**: DNS 解析 dig/getent 双重兜底

### 2.3 性能需求

- **NFR-PERF-01**: MCP Tool 调用超时 10 秒 (`subprocess.run(timeout=10)`)
- **NFR-PERF-02**: LLM 纯 CPU 可运行 (Ollama + Qwen3 4B，无 GPU 要求)
- **NFR-PERF-03**: 最小内存需求 8GB (含系统开销)
- **NFR-PERF-04**: SSE 流式响应延迟 <500ms (首 token 到达时间)
- **NFR-PERF-05**: Dashboard 数据刷新间隔 30 秒

### 2.4 可维护性需求

- **NFR-MAINT-01**: MCP 插件注册统一 (`MCPPluginRegistry` + `_auto_register_all`)
- **NFR-MAINT-02**: 健康评分配置文件化 (`health_score_config.json`，代码与配置分离)
- **NFR-MAINT-03**: 插件热加载 (`importlib.import_module`，插件加载失败不影响其他 Tool)

## 3. 约束条件

| 约束项 | 说明 |
|-------|------|
| 开发语言 | 后端: Python 3.13+ (FastAPI)；前端: TypeScript (Vue 3 + Vite) |
| LLM 提供商 | 本地 Ollama (Qwen3 4B 默认) / DeepSeek / OpenAI 兼容 API |
| 数据库 | SQLite (无需外部依赖，单文件部署) |
| 部署环境 | 麒麟 V10 / V11 (x86_64 + LoongArch)，兼容通用 Linux |
| MCP 协议 | 自研 Plugin 系统 (MCPTool + MCPPluginRegistry)，参数通过 inputSchema JSON Schema 校验 |
| 安全原则 | 最小权限 (Least Privilege) + 纵深防御 (Defense in Depth) + 默认拒绝 (Default Deny) |
| 命令执行约束 | 固定参数列表，禁止字符串拼接；白名单模式 (23 条命令)；高危参数词边界拦截 |
