## 3. 一键部署 (推荐)

### 3.1 交互式部署

```bash
cd SRE-agent
bash scripts/deploy.sh
```

脚本将依次完成：
1. 自动检测操作系统（麒麟/Ubuntu/Debian），适配 dnf/apt
2. 安装系统依赖（Python 3.13+ / Node.js 18+）
3. 创建 Python 虚拟环境 + 安装所有后端依赖
4. 构建前端（npm install + npm run build）
5. **LLM 交互配置** — 选择 Ollama 本地 或 DeepSeek 云端
6. 生成 `.env` 配置文件 + 创建 sreagent 低权限用户
7. 可选注册 systemd 服务（开机自启）
8. 部署验证（Tool 数量 / 安全护栏 / 前端构建 / 磁盘空间）

### 3.2 非交互式部署（CI/CD / 容器初始化）

```bash
# 使用 Ollama 本地模式（默认）
bash scripts/quick-deploy.sh

# 使用 DeepSeek 云端 API
bash scripts/quick-deploy.sh deepseek YOUR_API_KEY
```

## 4. 麒麟操作系统手动部署

### 4.1 环境准备

```bash
# 确认系统版本
cat /etc/os-release
# 麒麟 V10 输出: ID="kylin" 或 PRETTY_NAME="Kylin V10 SP3"
# 麒麟 V11 输出: ID="kylin" 或 PRETTY_NAME="Kylin V11"

# 确认架构 (龙芯需特别注意)
uname -m
# 预期: x86_64 或 loongarch64
```

#### 麒麟 dnf 包名速查

| 通用包名 | 麒麟 dnf 包名 | 说明 |
|----------|--------------|------|
| python3.13 | `python3.13` | Python 3.13 解释器 |
| python3.13-devel | `python3.13-devel` | psutil 编译依赖 |
| nodejs | `nodejs` | Node.js 运行时 |
| npm | `npm` | Node 包管理器 |
| gcc | `gcc` | C 编译器 (编译 psutil) |
| nftables | `nftables` | 麒麟 V10+ 默认防火墙 |
| docker-ce | `docker-ce` | 容器运行时 (可选) |

```bash
# 安装系统依赖
sudo dnf install -y python3.13 python3.13-devel gcc
sudo dnf install -y nodejs npm

# 龙芯 (LoongArch64) 额外注意:
# 1. 确认使用龙芯架构专用源
# 2. psutil 需从源码编译, 确保 gcc 已安装
# 3. 部分 Python 包可能无预编译 wheel, pip 会自动从源码构建
```

### 3.2 后端部署

```bash
cd backend

# 创建虚拟环境
python3.13 -m venv .venv
source .venv/bin/activate

# 安装依赖
pip install -r requirements.txt

# 配置 LLM (根据实际情况选择)
cp .env.example .env
vim .env
# LLM_PROVIDER=ollama          # 本地 Ollama
# LLM_BASE_URL=http://localhost:11434
# LLM_MODEL=qwen3:4b
# --- 或 ---
# LLM_PROVIDER=deepseek        # DeepSeek 云端
# LLM_BASE_URL=https://api.deepseek.com
# LLM_MODEL=deepseek-v4-flash
# LLM_API_KEY=sk-xxx
# --- 或 ---
# LLM_PROVIDER=qwen            # 通义千问云端
# --- 或 ---
# LLM_PROVIDER=openai          # OpenAI 兼容 API

# 创建专用低权限运行用户 (安全最佳实践)
sudo useradd -r -s /bin/false -d /nonexistent -M sreagent

# 数据库初始化 (首次运行自动建表)
mkdir -p data

# 启动 (生产模式)
uvicorn app.main:app --host 0.0.0.0 --port 8001 --workers 4
```

### 3.4 LLM 本地部署 (可选)

### 3.5 Nginx 反向代理

```nginx
# /etc/nginx/conf.d/sre-agent.conf
upstream sre_backend {
    server 127.0.0.1:8001;
    keepalive 64;
}

server {
    listen 80;
    server_name sre-agent.local;

    # 日志
    access_log /var/log/nginx/sre-agent-access.log;
    error_log  /var/log/nginx/sre-agent-error.log;

    # 前端静态文件
    location / {
        root /var/www/html/sre-agent;
        index index.html;
        try_files $uri $uri/ /index.html;

        # 静态资源缓存
        location ~* \.(js|css|png|jpg|svg|woff2)$ {
            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 代理
    location /api/ {
        proxy_pass http://sre_backend;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;

        # SSE 流式响应 (核心配置)
        proxy_buffering off;
        proxy_cache off;
        proxy_read_timeout 300s;
        proxy_http_version 1.1;
        chunked_transfer_encoding on;

        # 禁用响应压缩确保 SSE 实时送达
        gzip off;
    }

    # 健康检查端点无需缓存
    location /health {
        proxy_pass http://sre_backend/health;
        proxy_cache off;
    }
}
```

### 3.6 systemd 服务文件

```ini
# /etc/systemd/system/sre-agent.service
[Unit]
Description=SRE-agent Backend Service
Documentation=https://github.com/LuviaSeraphina/SRE-agent
After=network.target network-online.target
Wants=network-online.target

[Service]
Type=simple
User=sreagent
Group=sreagent
WorkingDirectory=/opt/sre-agent/backend
EnvironmentFile=/opt/sre-agent/backend/.env
ExecStart=/opt/sre-agent/backend/.venv/bin/uvicorn app.main:app \
    --host 127.0.0.1 \
    --port 8001 \
    --workers 4 \
    --log-level info
ExecReload=/bin/kill -HUP $MAINPID
Restart=on-failure
RestartSec=5s
KillMode=mixed
KillSignal=SIGTERM
TimeoutStopSec=30s

# 安全加固
NoNewPrivileges=yes
ProtectSystem=strict
ProtectHome=yes
ReadWritePaths=/opt/sre-agent/backend/data
ReadOnlyPaths=/proc /sys /var/log
PrivateTmp=yes
PrivateDevices=no
CapabilityBoundingSet=CAP_NET_RAW CAP_SYS_PTRACE
AmbientCapabilities=CAP_NET_RAW CAP_SYS_PTRACE

# 资源限制
LimitNOFILE=65536
LimitNPROC=4096
MemoryMax=2G

[Install]
WantedBy=multi-user.target
```

```bash

sudo systemctl daemon-reload
sudo systemctl enable sre-agent
sudo systemctl start sre-agent

# 查看状态

sudo journalctl -u sre-agent -f
```

## 5. Docker Compose 部署

### 4.1 Dockerfile

```dockerfile
# backend/Dockerfile
FROM python:3.13-slim

RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# 安全: 创建非 root 用户
RUN useradd -r -s /bin/false -d /nonexistent -M sreagent

WORKDIR /app/backend

# 依赖分层缓存
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# 源码
COPY . .

# 数据目录
RUN mkdir -p data && chown -R sreagent:sreagent data

USER sreagent
EXPOSE 8001

HEALTHCHECK --interval=30s --timeout=3s --start-period=10s \
    CMD python -c "import httpx; httpx.get('http://localhost:8001/health')" || exit 1

CMD ["uvicorn", "app.main:app", "--host", "0.0.0.0", "--port", "8001"]
```

### 4.2 docker-compose.yml

```yaml
version: "3.9"

services:
  backend:
    build:
      context: ./backend
      dockerfile: Dockerfile
    container_name: sre-agent-backend
    restart: unless-stopped
    ports:
      - "127.0.0.1:8001:8001"
    environment:
      - LLM_PROVIDER=${LLM_PROVIDER:-ollama}
      - LLM_BASE_URL=${LLM_BASE_URL:-http://ollama:11434}
      - LLM_MODEL=${LLM_MODEL:-qwen3:4b}
      - LLM_API_KEY=${LLM_API_KEY:-}
      - MAX_RISK_LEVEL=${MAX_RISK_LEVEL:-restricted}
      - REQUIRE_CONFIRMATION=${REQUIRE_CONFIRMATION:-true}
      - AUDIT_ENABLED=${AUDIT_ENABLED:-true}
      - DATABASE_URL=sqlite+aiosqlite:///data/state_store.db
    volumes:
      - sre-agent-data:/app/backend/data
      - /var/log:/host-logs:ro
      - /proc:/host-proc:ro
      - /sys:/host-sys:ro
    networks:
      - sre-agent-net
    cap_add:
      - NET_RAW        # ss 命令需要
      - SYS_PTRACE     # 进程检查需要
    security_opt:
      - no-new-privileges:true
    read_only: false
    tmpfs:
      - /tmp:size=64M

  frontend:
    build:
      context: ./frontend
      dockerfile: Dockerfile
    container_name: sre-agent-frontend
    restart: unless-stopped
    ports:
      - "80:80"
    depends_on:
      - backend
    networks:
      - sre-agent-net

  ollama:
    image: ollama/ollama:latest
    container_name: sre-agent-ollama
    restart: unless-stopped
    ports:
      - "127.0.0.1:11434:11434"
    volumes:
      - ollama-data:/root/.ollama
    networks:
      - sre-agent-net
    profiles:
      - local-llm       # 仅在使用本地 LLM 时启用
    deploy:
      resources:
        reservations:
          devices:
            - driver: nvidia
              count: 1
              capabilities: [gpu]

networks:
  sre-agent-net:
    driver: bridge

volumes:
  sre-agent-data:
  ollama-data:
```

## 8. 安全配置建议

### 7.1 最小权限

```bash
# 1. 创建专用运行用户 (必须, 不可省略)
sudo useradd -r -s /bin/false -d /nonexistent -M sreagent

# 2. 配置 sudo 策略 (最小化原则)
sudo visudo -f /etc/sudoers.d/sre-agent
# 仅授予必要命令:
# sreagent ALL=(root) NOPASSWD: /usr/bin/systemctl status *-*, /usr/bin/journalctl *
# sreagent ALL=(root) NOPASSWD: /usr/bin/docker ps, /usr/bin/podman ps
# sreagent ALL=(ALL) NOPASSWD: /usr/bin/crontab -l *

# 3. 禁止 shell 登录
sudo usermod -s /sbin/nologin sreagent
```

### 7.2 日志保护

```bash
# 审计日志仅追加
sudo chmod 640 /var/log/auth.log /var/log/secure
sudo chattr +a /var/log/auth.log   # 仅追加 (需 root)
```

### 7.3 网络隔离

```bash
# 生产环境仅监听本地回环
# systemd: --host 127.0.0.1  (见 §3.6)
# docker:  ports: "127.0.0.1:8001:8001" (见 §4.2)
# 通过 nginx 反向代理对外暴露
```

### 7.4 `.env` 安全

```bash
# 权限收紧

# 敏感变量
```

## 9. 故障排除

| 现象 | 原因 | 解决 |
|------|------|------|
| `psutil` 安装失败 | 缺少 `python3-devel` / `gcc` | `dnf install python3.13-devel gcc` |
| `journalctl` 不可用 | 非 systemd 系统 | 自动降级到日志文件读取 |
| 龙芯架构 Tool 返回空 | psutil 版本过低 | `pip install psutil>=5.9 --upgrade` |
| 龙芯 `loongarch64` pip 包缺失 | 部分包无预编译 wheel | 系统自动从源码编译 (需 gcc) |
| 前端构建 OOM | Node 内存不足 | `NODE_OPTIONS="--max-old-space-size=4096" npm run build` |
| SSE 连接断开 | nginx 缓冲超时 | 配置 `proxy_buffering off; proxy_read_timeout 300s;` (见 §3.5) |
| `docker` 命令无权限 | 非 root 运行 | `usermod -aG docker sreagent` 或授 sudo |
| `aa-status` 工具不可用 | 非 AppArmor 系统 | `security_selinux_status` 自动降级到 `getenforce` |
| SQLite 数据库锁定 | 并发写入冲突 | SQLite 适用单机部署; 多副本部署请切换 PostgreSQL |
| Ollama 连接失败 | 服务未启动或地址错误 | 检查 `LLM_BASE_URL`, 确认 `ollama serve` 已运行 (麒麟系统需确认端口: `ss -tlnp | grep 11434`) |
