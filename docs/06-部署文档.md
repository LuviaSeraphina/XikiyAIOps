# 部署文档

> 版本：v1.2.0 | 最后更新：2026-07-11

---

## 1. 环境要求

- Linux（Debian / Ubuntu / 麒麟 V10+）
- Debian/Ubuntu 用 DEB 包，麒麟用 tar.gz（麒麟无 dpkg）
- Python 3.11+、systemd
- 2 核 CPU / 2 GB 内存 / 5 GB 磁盘

---

## 2. DEB 包安装（Debian / Ubuntu）

### 2.1 安装

```bash
sudo dpkg -i xikiy-aiops_1.2.0_amd64.deb
```

安装自动完成：Python 虚拟环境 → 离线依赖 → 数据库初始化 → systemd 服务注册 → Prometheus 部署。

### 2.2 验证

```bash
curl http://localhost:8001/health
# → {"status":"ok","service":"xikiy-aiops","llm":"ok"}
```
### 2.3 打开前端

浏览器打开 `http://localhost:8001`

### 2.4 配置LLM

打开前端界面后, 侧边栏 → **模型配置**，选择预设模型并输入 API Key，点击「保存配置并使用」。

### 2.5 CLI 命令

```bash
xikiy-aiops start      # 启动
xikiy-aiops stop       # 停止
xikiy-aiops restart    # 重启
xikiy-aiops status     # 状态
xikiy-aiops logs       # 日志（默认 50 行）
xikiy-aiops audit      # 审计日志
```

> tar.gz 安装无 CLI 快捷命令，直接用 `systemctl`（见 3.5 节）。

---

## 3. tar.gz 安装包（通用 Linux / 麒麟）

### 3.1 安装

```bash
# 1. 解压到任意目录（桌面、/tmp、/opt 均可）
tar xzf XikiyAIOps_v1.2.0_loongarch64.tar.gz
cd XikiyAIOps_v1.2.0

# 2. 一键部署（自动安装到 /opt/xikiy-aiops/）
sudo bash scripts/deploy.sh

# 3. 启动服务
bash /opt/xikiy-aiops/scripts/start.sh
```

`deploy.sh` 自动完成：创建 xikiy 最小权限用户 → Python 虚拟环境 + 离线依赖 → .env 生成 → 数据库初始化 → 免密 sudo 白名单 → systemd 服务注册。

- 桌面上的解压目录和安装包部署完可以删掉。

### 3.2 验证

```bash
curl http://localhost:8001/health
# → {"status":"ok"}
```

### 3.3 打开前端

浏览器打开 `http://localhost:8001`

### 3.4 配置LLM

打开前端界面后, 侧边栏 → **模型配置**，选择预设模型并输入 API Key，点击「保存配置并使用」。

### 3.5 启停

```bash
systemctl start xikiy-aiops     # 启动
systemctl stop xikiy-aiops      # 停止
systemctl restart xikiy-aiops   # 重启
systemctl status xikiy-aiops    # 状态
journalctl -u xikiy-aiops -f    # 日志
```

## 4. 安全模型

### 4.1 最小权限代理

Agent 进程以专用用户 `xikiy` 运行（systemd `User=xikiy`），不直接使用 root：

| 用户 | 用途 | 组成员 |
|------|------|--------|
| `xikiy` | Agent 运行时用户 | `sudo`/`wheel`（sudo 白名单），`systemd-journal`/`adm`（读日志） |
| 部署用户 | 初始安装者 | `xikiy`（可管理 xikiy 拥有的文件） |

### 4.2 sudo 白名单

`/etc/sudoers.d/xikiy-aiops` 授予 `%xikiy` 组 15 个命令的 NOPASSWD 权限：`systemctl`、`truncate`、`logrotate`、`apt`/`dnf`、`sysctl`、`renice`、`ionice`、`kill` 等。

### 4.3 日志读取

`xikiy` 加入 `systemd-journal` 和 `adm` 组，可直接读取 journalctl 和 /var/log，无需 sudo。

## 5. 目录结构

```
/opt/xikiy-aiops/
├── backend/
│   ├── app/              # 应用代码
│   ├── .venv/            # Python 虚拟环境
│   ├── data/             # 数据库 + RAG 知识库
│   └── .env              # 运行配置
├── frontend/dist/        # 前端构建产物
├── prometheus/           # 监控配置
└── offline-packages/     # 离线 wheel
```
