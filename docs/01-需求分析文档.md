# 需求分析文档

> 第十五届中国软件杯大赛 A 组赛题 — 面向麒麟操作系统的安全智能运维 Agent
> 出题企业: 麒麟软件有限公司

---

## 1. 赛题分析

### 1.1 核心命题
构建部署于麒麟操作系统之上的智能运维 Agent，通过 MCP 协议赋予大模型感知系统状态、执行运维管理任务的能力。

### 1.2 评分权重分析

| 评分项 | 权重 | 本项目对应实现 |
|--------|:----:|---------------|
| 安全护栏 | 20% | 三层护栏: 意图过滤 + 注入检测 + 权限代理 |
| 功能完整性 | 40% | 21 个 MCP Tool + 根因分析 + 仪表盘 |
| 麒麟 OS 适配 | 15% | 自动平台检测 + dnf/nftables/龙芯兼容 |
| 智能化程度 | 15% | LLM 驱动 + 异常检测 + 因果推断 |
| 文档与演示 | 10% | 5 份完整文档 + 部署指南 |

---

## 2. 功能需求

### 2.1 系统感知 (MCP Tools — 21 个)

#### 进程感知
- FR-PROC-01: 进程列表查看，支持按 CPU/内存/状态排序
- FR-PROC-02: 进程状态过滤 (running/sleeping/zombie/stopped)

#### 磁盘感知
- FR-DISK-01: 磁盘使用率查询，支持指定挂载点

#### 网络感知
- FR-NET-01: 监听端口审计 (端口/协议/进程/PID)
- FR-NET-02: TCP 连接状态统计 (ESTABLISHED/CLOSE_WAIT/TIME_WAIT)
- FR-NET-03: 网卡流量统计 (RX/TX 字节/包/错误/丢包)

#### 内存感知
- FR-MEM-01: 物理内存使用画像 (总量/已用/可用/百分比)
- FR-MEM-02: Swap 使用率监测
- FR-MEM-03: OOM Killer 历史追溯

#### 安全态势 (核心亮点)
- FR-SEC-01: 多源认证失败统计 (SSH/su/sudo/PAM/fail2ban)
- FR-SEC-02: 活跃会话与 SSH 连接枚举
- FR-SEC-03: SUID/SGID 后门文件扫描
- FR-SEC-04: 用户 crontab 持久化审计
- FR-SEC-05: 内核模块审计 (Rootkit 检测)
- FR-SEC-06: 安全更新检测 (dnf/apt 自动适配)
- FR-SEC-07: 用户权限审计 (空密码/UID=0/NOPASSWD)
- FR-SEC-08: 内核安全参数基线审计 (12 项)
- FR-SEC-09: 用户与组查询

#### 系统健康
- FR-SYS-01: 系统概览 (内核/发行版/架构/启动时间)
- FR-SYS-02: 负载平均值 (1/5/15min) + 过载判断
- FR-SYS-03: 失败 systemd 服务检测
- FR-SYS-04: 内核启动参数安全审计

### 2.2 安全护栏 (3 层 6 机制)

#### 第一层: 意图风险过滤
- FR-SAFE-01: 四层递进分类 (越狱→高危→注入→运维)
- FR-SAFE-02: 正则词边界匹配 + 组合加权评分 (v2.0)
- FR-SAFE-03: 中英文双语越狱模式检测
- FR-SAFE-04: 威胁评分 (0.0~1.0) + 风险等级 (CRITICAL→SAFE)

#### 第二层: Prompt 注入检测
- FR-SAFE-05: 分隔符注入检测 (\n/\r/\x00)
- FR-SAFE-06: 编码绕过检测 (Base64/Hex)
- FR-SAFE-07: 间接指令注入检测 (翻译嵌套/规则绕过)
- FR-SAFE-08: Unicode 同形字混淆检测 (v2.0)
- FR-SAFE-09: 零宽字符隐蔽注入检测 (v2.0)
- FR-SAFE-10: 大小写交替混淆检测 (v2.0)
- FR-SAFE-11: LLM 输出黑名单二次校验

#### 第三层: 最小权限代理
- FR-SAFE-12: 风险等级 → 权限等级映射 (read_only/restricted/dangerous)
- FR-SAFE-13: sudo 降权执行机制 (v2.0)
- FR-SAFE-14: 专用低权限用户 (sreagent) 支持 (v2.0)
- FR-SAFE-15: 命令白名单 + 高危参数拦截
- FR-SAFE-16: 保护进程/路径名单 (即使 root 也不可操作)

### 2.3 智能化根因分析

- FR-RCA-01: 3-Sigma 异常检测
- FR-RCA-02: IQR 异常检测
- FR-RCA-03: 移动平均异常检测
- FR-RCA-04: 多算法组合投票
- FR-RCA-05: 日志事件结构化提取
- FR-RCA-06: 因果关联模式匹配 (OOM/磁盘/网络/CPU/认证)
- FR-RCA-07: 因果链推断
- FR-RCA-08: 系统健康度综合评分 (0~100)

### 2.4 前端交互

- FR-UI-01: 对话式交互 (SSE 流式输出)
- FR-UI-02: 系统仪表盘 (CPU/内存/磁盘/安全态势)
- FR-UI-03: 审计日志时间线
- FR-UI-04: Tool 调用可视化卡片
- FR-UI-05: 高危操作确认对话框

## 3. 非功能需求

### 3.1 安全需求
- NFR-SEC-01: AI 永远不直接接触 Shell
- NFR-SEC-02: 所有命令执行必须通过 MCP Tool 层
- NFR-SEC-03: 高危操作必须记录审计日志
- NFR-SEC-04: 命令参数必须是固定列表, 禁止字符串拼接

### 3.2 兼容性需求
- NFR-COMPAT-01: 支持麒麟 V10 (x86_64 + 龙芯)
- NFR-COMPAT-02: 支持麒麟 V11
- NFR-COMPAT-03: 兼容 Ubuntu 22.04 / Debian 12
- NFR-COMPAT-04: 自动检测 dnf/apt 包管理器
- NFR-COMPAT-05: 自动检测 nftables/iptables 防火墙

### 3.3 性能需求
- NFR-PERF-01: Tool 调用超时 10 秒
- NFR-PERF-02: LLM 纯 CPU 可运行 (Ollama + Qwen3 4B)
- NFR-PERF-03: 最小内存需求 8GB

---

## 4. 约束条件

1. 开发语言: Python 3.13+ (后端) / TypeScript + Vue 3 (前端)
2. LLM: 支持本地部署 (Ollama) 和远程 API
3. 部署环境: 麒麟 V10/V11 操作系统
4. 安全原则: 最小权限 + 纵深防御
